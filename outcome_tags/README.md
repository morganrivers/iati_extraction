# outcome_tags

Binary tags applied to each activity describing what happened, used as prediction targets.

## Files

### `applied_tags.jsonl` (1,800 rows)
One record per activity. Tags were generated by an LLM reading the ex-post evaluation document and are used to train classifiers that predict outcomes from ex-ante features.

**Fields:**
- `activity_id` — IATI activity identifier
- `unsigned_tags` — list of tags that describe factual events or conditions where if not mentioned in the summary, assumed to not occur (e.g. `"funds_cancelled_or_unutilized"`, `"external_factors_affected_outcomes"`, `"activities_not_completed"`)
- `signed_tags` — list of tags that carry an explicit success/failure signal where a negative result is required, otherwise treated as not having information (e.g. `"capacity_building_delivered"`, `"over_budget"`, `"infrastructure_completed"`, `"targets_met_or_exceeded"`)
