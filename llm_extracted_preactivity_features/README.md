# llm_features

Raw LLM outputs used to derive features for the forecasting model. Most files have ~2,100–2,200 rows covering the full activity set (train + val + test + extra activities graded for KNN context).

All JSONL files use `activity_id` as the primary key.

## Common fields
- `activity_id` — IATI activity identifier
- `section` — which document section the prompt was applied to (typically `"Baseline"` for ex-ante data; `None` if not applicable)
- `model_version` — LLM used to generate the response
- `response_text` — raw JSON string returned by the LLM

---

## Files

### `outputs_*_grades.jsonl`
LLM-assigned rubric scores for specific evaluation dimensions. Each file corresponds to one dimension; `response_text` is a JSON object with the numeric score and reasoning.

- `outputs_complexity_grades.jsonl` — how complex the activity design is
- `outputs_context_grades.jsonl` — quality of contextual/situational analysis in project documents
- `outputs_finance_grades.jsonl` — quality of the financial plan and cost-effectiveness
- `outputs_implementer_performance_grades.jsonl` — track record and capacity of implementing organisations
- `outputs_integratedness_grades.jsonl` — how well the activity is integrated with other programmes/stakeholders
- `outputs_risks_grades.jsonl` — quality of risk identification and mitigation
- `outputs_targets_grades.jsonl` — clarity and measurability of targets and indicators

### `outputs_targets.jsonl`
LLM extraction of quantitative targets stated in the activity document (baseline, target, unit, indicator name). `response_text` is a JSON array of target objects.

### `outputs_misc.jsonl`
Miscellaneous LLM-extracted fields not covered by the rubric grades (e.g. activity summary, country/sector classification, implementer names). `response_text` is a structured JSON object. `num_pages` records how many PDF pages were processed.

### `outputs_finance_sectors_disbursements_baseline_gemini2p5flash.jsonl`
LLM extraction of financial allocation by sector and outcome type (grant vs. loan, amount in USD). Generated by Gemini 2.5 Flash. `prompt_type` distinguishes sub-prompts if multiple were used.

### `outputs_targets_context_maps_trainval.jsonl`
Embedding-derived context features. UMAP was refit on train+val combined (for use when evaluating the test set). One row per activity.

**Key fields:**
- `decade` / `decade_label` — temporal cohort of the activity start date
- `sector` — high-level sector string
- `countries` / `country_weights` — list of recipient countries and their weight (by disbursement share)
- `sector_distance` — cosine distance to the centroid of same-sector activities in embedding space
- `country_distance` — cosine distance to the centroid of same-country activities
- `dropped_country_weight` — weight of countries that were dropped (e.g. "GLOBAL") before computing distance
- `umap_2d` / `umap_3d` / `umap_4d` — UMAP-compressed embedding coordinates (lists of 2, 3, 4 floats)

### `llm_planned_duration.jsonl`
LLM-extracted planned activity duration.
- `planned_duration_years` — duration in years (float)
- `source` — which source (IATI planned duration, or in other cases, the LLM extracted planned duration which was preferred to minimize leakage)

### `llm_planned_expenditure.jsonl`
LLM-extracted planned total expenditure.
- `planned_expenditure_usd` — amount in USD (float)
